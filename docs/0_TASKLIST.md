# Lore Weaver - 交互式多 Agent 架构细化升级计划 (Step 1.1 ~ Step 3.3)

## 📌 架构设计：神识共鸣 (Human-in-the-Loop) 多 Agent 协同体
本系统重构成 **“五部典仪子 Agent”** 深度共鸣会商架构，通过多段式“暂挂与神识调整”机制运作：
1. **世界编制官 (`WorldBuilderAgent` | Step 1.1)**: 主理 IP DNA 元数据、3-Resource (灵、魂、魄) 经济杠杆、6-12 重幻境阶梯等级。
2. **剧本大纲师 (`NarrativeAgent` | Step 1.2)**: 主理 12 重剧情大纲、关卡锁条件、失败惩罚与星级成就数值规划。
3. **沙盒架构师 (`SandboxArchitect` | Step 2.1 - 2.2)**: 主理 H5 宿主包解耦部署、ScaleFIT 分辨率强制锁、数据库 `store.js` 及寄存器注入。
4. **代码铸造厂 (`CodeFoundryAgent` | Step 3.1 - 3.2)**: 主理 12 关物理逻辑代码合成、ASMR 视听、VFX 特效漂浮词条及文字换行包裹。
5. **多模审计官 (`AuditorAgent` | Step 3.3)**: 主理 E2E 与 VLM 多模态像素碰撞诊断，去除敏感 IP 并沉淀回流全局优化知识库。

---

## 第 1 阶段：创意与数值设计空间（可交互微调段）
- [x] 1.1 **[Step 1.1: IP DNA 与 3-Resource 经济编制]** `[WorldBuilderAgent]`：
  - **模型交互**：后端支持提取与生成 IP DNA Assertions 及境界体系。
  - **神识会商**：前端支持针对此步骤产出的经济学模型和特征，与 **“世界编制官”** 展开多轮神识共鸣对话，提供微调指令（如“降低天材地宝产出，灵力消耗调高 20%”），重新生成。
- [x] 1.2 **[Step 1.2: 12 重天主线剧本与关卡契约规划]** `[NarrativeAgent]`：
  - **大纲生成**：依据 Step 1.1 所定义的经济世界元数据，自动衍生 12 Node 精密主线剧情。
  - **方案合并 (GDD/PRD)**：融汇 1.1 与 1.2 的成果，首创常驻 **作品设计方案 (GDD/PRD)** 自定义文档，支持直接预览。
  - **神识会商**：提供专门对于大纲剧情、惩罚规则的微调对话，等用户全面赞同 "Approve" 确认后，方才释放进入代码物理编译阶段。

## 第 2 阶段：工程架设与物理生命周期定义（全自动/热补丁段）
- [x] 2.1 **[Step 2.1: 系统宿主脚手架精密部署]** `[SandboxArchitect]`：
  - **部署落地**：一键生成并在工作物理沙盒路径自动部署标准 `package.json`, `vite.config.ts` 以及 Phaser 入口代码 `js/main.js`。
  - **视口配置**：固定 ScaleFIT (720x1280) 移动端完美锁定分辨率，保障视觉无变形。
- [x] 2.2 **[Step 2.2: 记忆数据库与状态寄存器注入]** `[SandboxArchitect]`：
  - **配方同步**：沙盒自动注入本地临时持久化类 `store.js` 接口与 `data.js` 静态注册表，预打包技能树及修仙天道装备合成配方。

## 第 3 阶段：物理关卡工厂与视听着色合成（全自动/热补丁段）
- [x] 3.1 **[Step 3.1: 12 关物理卡牌主循环铸造]** `[CodeFoundryAgent]`：
  - **高保真落库**：系统根据 12 重剧情大纲的关卡契约，用物理卡牌机制生成 12 重独立的 JavaScript 编译副本。
  - **无损控制**：严格注入 `transitionLock` 与 `shutdown` 物理注销和垃圾回收钩子，防止重载时内存泄露引发的 Canvas 卡死现象。
- [x] 3.2 **[Step 3.2: Web Audio ASMR 调制与特效色相]** `[CodeFoundryAgent]`：
  - **ASMR 调制**：内置 Web Audio 合成器配平风暴/雷击声学振荡。
  - **屏幕震感/漂浮词条**：设计高振动碰撞感、战斗伤害漂浮词条 (VFX Floaters) 与自适应中文字换行包裹函数。
- [x] 3.3 **[Step 3.3: 多模态审计质量检修与知识回流]** `[AuditorAgent]`：
  - **编译合规检测**：通过 VLM 多模态智能检查，校准碰撞箱重叠并测试游戏循环。去敏感化后将错误经验回执沉淀至全局 Prompts 库。

## 第 4 阶段：人机协同与多 Agent 极客 UI 面板重构
- [x] 4.1 **[全景式 Timeline 齿轮管线面板]**：
  - 重塑左侧进度展示，改为精美的 **1.1 ~ 3.3 全链纵向流动轨迹**，实时指示当前的编译活跃步骤及百分比。
- [x] 4.2 **[神识共鸣 HITL Chamber 会商窗口]**：
  - 新增独立交互式会商卡。下拉选单支持快捷切换至特定的 **五部典仪子 Agent**（如剧本大纲师、ASMR 视听调音师），可针对某一特定步骤进行细化对话与代码打补丁热修复。
- [x] 4.3 **[保持 GDD/PRD 设计文档不被冲散]**：
  - 完美保障 GDD 展示视窗的优越性，作为 1.1 与 1.2 的高光产出物，常驻为用户的设计大本营。

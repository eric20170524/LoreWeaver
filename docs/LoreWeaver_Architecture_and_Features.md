# LoreWeaver 多 Agent 拟真协同管线：系统架构与核心功能技术白皮书

LoweWeaver 是一个高度精细化、面向移动端 H5 锁屏微游的同人小说 IP 智能转译与物理编译生产系统。本系统基于“人机协同神识共鸣 (Human-in-the-Loop)”的设计原语，将大型语言模型 (LLM) 转化为具备特定工程、视听与合规学识的五部典仪子 Agent，形成紧密闭环。

---

### 1.1 服务端分层设计
- **API 编排层 (backend/main.py)**: 负责维护异步长任务管线 (BackgroundTasks)、会商对话寻址中转以及基于工作区 (Workspace) 的临时状态生命周期。
- **Agent 本能层 (backend/agents.py)**: 承载具有特定指令深度包装的 LLM。按领域逻辑区隔，并在生成阶段（1.1/1.2）及微调应用阶段（adjust）针对特定字段自适应寻优。
- **数据持久层 (backend/database.py & SQLite)**: 保障工作区元数据、配置清单 JSON、以及调试日志的幂等保存与序列化恢复。

### 1.2 客户端排版美学
- **Slate Dark 暗色宇宙美学**: 极简深灰 `bg-slate-950` 背景辅以高对比度 `emerald` 翠色发光元器件，为极客式神识会商提供安静、清晰的工作面。
- **自适应视口锁屏器 (720x1280 Mobile Chassis)**: 外挂圆角 phone 模拟边框、顶部听筒胶囊与背景发光漫反射（Aura Gloom Glow），完美呈现移动沙盒真实像素。

## ── 2. 五部典仪 Agent 核心功能 (Core Features) ──

### 2.1 世界编制官 (WorldBuilderAgent | Step 1.1)
- **功能描述**: 将原始修仙/末世/玄幻同人词条逆向解析为底层 IP DNA 因子。
- **参数控制**:
  - `themeColor`: 游戏的主视视觉十六进制色彩。
  - `economy`: 锁定 `Qi` (气/功法)、`Bones` (骨/境界突破资源)、`Essence` (魄/天材地宝) 三大核心资源因子循环杠杆。
  - `realms`: 定义神骸飞升的 6 大修行阶梯，生成前后端完全统一的文字模。

### 2.2 剧本大纲师 (NarrativeAgent | Step 1.2)
- **功能描述**: 基于 Step 1.1 决议的经济参数，自动扩展生成 12 重关卡支点的剧情神迹树。
- **参数控制**:
  - `title` / `intro`: 每一关独立的修仙宿命剧情词条及 BOSS 嘲讽文本。
  - `action`: 生成卡牌玩法的底层卡牌指令（如 “Tap” 顿悟、“Dodge” 避劫、“Memory” 意念匹配）。
  - `levelDifficulty`: 精算随境界螺旋上升的触发几率与耗时限制。
  - 产出 **作品设计方案 (GDD/PRD)** 固化清单并提供常驻文档，供用户审阅。

### 2.3 沙盒架构师 (SandboxArchitect | Step 2.1 - 2.2)
- **功能描述**: 将数值方案转交至工程脚手架，实现物理运行环境初始化。
- **参数控制**:
  - `package.json` / `vite.config.ts`: 部署极速编译所需标准模块结构。
  - `store.js` / `data.js`: 装配状态机，强制寄存并同步当前解锁卡片进度和个人修仙代币，支持持久化存储。

### 2.4 代码铸造厂 (CodeFoundryAgent | Step 3.1 - 3.2)
- **功能描述**: 将卡片数值完全挂载至 Phaser 3 Canvas，注入 ASMR 谐振。
- **参数控制**:
  - `Phaser.Game` 物理卡牌微循环绑定。
  - `Web Audio API` 脉冲声学配平：配置锯齿波及滤波共鸣，为玩家动作引入天雷震动等 ASMR 谐算。
  - `VFX Floaters`: 特效字浮空上升、中文字模自适应换行，确保长文本在 720x1285 容器中不穿模。

### 2.5 多模审计官 (AuditorAgent | Step 3.3)
- **功能描述**: 发挥多模态大语言模型 VLM (Visual Language Model) 的像素定位特性，审计游戏排版、文字覆盖与视口碰撞。
- **参数控制**:
  - 去除一切侵权敏感词沉淀。
  - 检测 HUD 边界重叠故障，并将诊断成功的数据落入基准库，向前端推送完美 [PASS] 诊断日志。

## ── 3. 人机共鸣交互 (HITL Chamber 协同会商) ──

- **子 Agent 快捷寻址**: 允许通过前端下拉菜单直接唤起五部特定 Agent，精确定向咨询。
- **热补丁反馈回路**:
  - 当在 1.1/1.2 编制阶段突发奇想，无需重写整个同人词，用户对特定编撰官发送细化描述（如“主标题增加国风重彩感”），大模型会在保留原 JSON 的安全字段前提下热合并并回显 PRD 文档。
  - 提供 `Approve & Merge`（确认无误，合成落库）的最终安全防护网，确保将纸面 PRD 完美送入代码落库段，彻底告别盲盒流生成。

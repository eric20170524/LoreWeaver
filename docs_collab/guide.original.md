让两个本地 AI 智能体（Agent）互为“架构师”和“开发工程师”进行协作，是一个非常前沿且高效的开发模式。为了让 codex 和 antigravity 顺畅协作，我们需要为它们设定明确的**角色边界**，并建立一个**状态流转机制**（比如通过共享文件或命令行上下文来传递信息）。
以下是为你量身定制的完整协作方案，包含严格的流程指南与每个环节的专属提示词（Prompt）。
## 整体协作流程指南
为了确保双方不会“各说各话”，必须让它们遵循严格的顺序。
 1. 阶段一：Codex 分析需求输出方案
   角色：架构师
   你（人类）提供原始需求，Codex 负责将其转化为结构化的《技术方案设计文档》，包括系统架构、数据流和核心接口。
 2. 阶段二：Antigravity 进行评审反馈
   角色：资深开发 / 挑战者
   Antigravity 以执行者的视角审阅方案，专门挑刺。寻找逻辑漏洞、实现难度过高的部分，以及可能遗漏的边界情况，并输出《评审意见》。
 3. 阶段三：Codex 吸收反馈并输出任务清单
   角色：项目经理
   Codex 阅读评审意见，修正原方案，并将工作拆解为颗粒度极细、符合 MECE（相互独立、完全穷尽）原则的《任务执行清单》。
 4. 阶段四：Antigravity 执行单个任务
   角色：主力开发
   Antigravity 严格按照《任务执行清单》，每次只选取排在最前面的一个未完成任务进行代码编写，完成后提交评审请求。
 5. 阶段五：Codex 循环评审与卡点
   角色：代码审查员 (Reviewer)
   Codex 对 Antigravity 提交的代码进行 Code Review。如果发现 Bug 或不符合规范，打回并要求重写；如果通过，则标记该任务完成，通知 Antigravity 推进下一个任务。
## 核心提示词与执行模板
请在对应的环节，将以下提示词喂给对应的 AI。建议在同一项目目录下建立一个 docs/ 文件夹，专门用于它们交换文档（如 design.md, feedback.md, tasks.md）。
### 1. Codex：分析需求输出方案
**输入给 Codex：**
```markdown
# Role
你现在的角色是资深软件架构师兼项目经理。

# Task
请阅读以下[原始需求]，为其设计一份详细的技术落地建议书。

# Requirements
你需要输出一份 Markdown 文档，包含以下模块：
1. 需求理解与核心目标
2. 系统架构设计（包含技术栈建议）
3. 核心功能模块划分
4. 关键数据结构或接口设计

请保持专业、严谨。准备好后，请输出你的方案，供开发工程师（Antigravity）评审。

[原始需求]：
（在这里粘贴你的具体项目需求）

```
### 2. Antigravity：进行评审反馈
**输入给 Antigravity：**
```markdown
# Role
你现在的角色是资深后端/前端开发工程师，你注重代码的可维护性、性能和落地可行性。

# Task
请阅读架构师（Codex）输出的[技术方案]，进行深度评审。

# Requirements
不要盲目赞同，你需要带着“挑刺”的眼光，重点检查：
1. 方案中是否存在技术难点或不可行的部分？
2. 是否有遗漏的边界情况（Edge Cases）或异常处理逻辑？
3. 数据结构或接口设计是否合理？
请列出具体的风险点，并给出你的修改建议。

[技术方案]：
（在这里粘贴 Codex 输出的方案文档，或让其读取对应文件）

```
### 3. Codex：阅读评审并输出任务清单
**输入给 Codex：**
```markdown
# Role
你依然是资深架构师兼项目经理。

# Task
请阅读开发工程师（Antigravity）提供的[评审意见]，修正你的设计思路，并将整个项目拆解为可执行的《任务清单》。

# Requirements
1. 简要说明你接受了哪些评审意见，并在心中更新设计。
2. 输出一份 Markdown 格式的《任务清单》。
3. 任务拆解必须遵循“原子化”原则，每个任务的完成时间应较短，且有明确的验收标准（Done Criteria）。
4. 使用复选框（- [ ] 任务名）格式，标明任务的先后依赖顺序。

[评审意见]：
（在这里粘贴 Antigravity 的反馈）

```
### 4. 循环开发阶段：Antigravity 执行任务
**输入给 Antigravity：**
```markdown
# Role
你是主力开发工程师。

# Context
这是我们项目的《任务清单》。我们将采取“单步执行-代码评审-继续”的循环模式。

# Task
请查看[任务清单]，找到**第一个未完成（即未打勾）的任务**。
1. 阅读该任务的要求，并编写完整的代码。
2. 确保代码包含必要的注释。
3. 完成后，输出你的代码实现，并附上一段简短的说明，向架构师申请 Code Review。

注意：每次只执行一个任务！不要越界开发后续任务。

[任务清单]：
（提供当前最新的任务清单状态）

```
### 5. 循环开发阶段：Codex 代码评审
**输入给 Codex：**
```markdown
# Role
你现在的角色是严苛的 Code Reviewer。

# Task
开发工程师（Antigravity）刚刚提交了[当前任务]的代码实现，请进行代码审查。

# Requirements
你的审查结果只能是以下两种情况之一：
- **[拒绝]**：指出代码中的 Bug、性能隐患、不符合方案设计的地方或缺失的逻辑。要求 Antigravity 重新修改，直到你满意为止。
- **[通过]**：如果代码质量达标，明确回复“评审通过”。更新并输出最新的《任务清单》（将刚才的任务打勾），并指示 Antigravity 开始执行下一个未完成的任务。

[当前任务描述]：...
[Antigravity提交的代码]：...

```
> **给人类（你）的建议：**
> 如果你是通过纯命令行脚本（比如 Python 脚本）来调度这两个 AI，你可以利用脚本自动抓取前一个模型的输出，并将其作为变量注入到下一个 Prompt 的 [] 占位符中。如果你是手动复制粘贴，确保它们每次都能看到最新的《任务清单》状态。
